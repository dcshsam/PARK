# SAP Proposal Review

Use this skill when evaluating SAP-related vendor proposals, RFP responses, statement-of-work drafts, or implementation roadmaps. The focus is on balancing business fit, technical feasibility, SAP-specific architecture, commercial terms, and delivery risk.

## Core Review Workflow

For every proposal review, follow these steps:

1. **Clarify scope and stakeholder priorities**
   - Identify the SAP domain: S/4HANA implementation/upgrade, BTP extension, RISE with SAP, integration, analytics, or managed services.
   - Confirm the buyer’s top priorities (e.g., speed to value, clean core, TCO, industry fit, risk reduction).
   - Note any mandatory requirements or must-have capabilities.

2. **Map proposal structure against a standard framework**
   - See [references/evaluation-framework.md](references/evaluation-framework.md) for the standard criteria and suggested weights.
   - Check whether the proposal clearly addresses each criterion. Flag missing or weak sections.

3. **Apply SAP-specific scrutiny**
   - See [references/sap-specific-checklist.md](references/sap-specific-checklist.md) for SAP-specific questions around clean core, extensibility, integration, migration approach, licensing, and support.

4. **Score and compare**
   - Use a 1–5 scale per criterion (1 = poor/missing, 5 = excellent).
   - Multiply by criterion weight and sum for a weighted score.
   - When comparing multiple proposals, use the same weights and reviewers to keep scoring consistent.

5. **Summarize findings**
   - Strengths, weaknesses, risks, assumptions, and open questions.
   - Recommended next steps (e.g., clarification requests, reference checks, demo scope, negotiation points).

## Output Format

Present the review in a concise structure:

- **Executive verdict** – Short recommendation (shortlist / reject / request clarification).
- **Compliance check** – Pass/fail against mandatory requirements.
- **Weighted scorecard** – Criteria, scores, weights, weighted totals.
- **SAP-specific assessment** – Clean core, integration, migration, licensing highlights.
- **Risk and assumptions** – Top risks, missing assumptions, commercial concerns.
- **Recommended actions** – What the buyer should do next.

## Common Red Flags

- Vague scope, milestones, or acceptance criteria.
- No clear data migration or cutover strategy.
- Heavy customization without a clean-core or BTP-extension rationale.
- Missing SAP licensing or indirect-access considerations.
- Fixed-price proposals with incomplete requirements or unclear change-control.
- No reference customers for the proposed scope or industry.
- No plan for hypercare, knowledge transfer, or steady-state support.
